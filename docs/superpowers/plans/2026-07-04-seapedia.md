# SEAPEDIA Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the SEAPEDIA multi-role marketplace (COMPFEST 18 challenge, Levels 1–7) per `docs/superpowers/specs/2026-07-04-seapedia-design.md`.

**Architecture:** Monorepo: `backend/` Express 5 + TypeScript + Prisma + SQLite API (layered routes→controllers→services), `frontend/` React 19 + Vite + TS + Tailwind v4 SPA. DB-backed sessions carry the active role; all business rules live in services; every stock/balance/quota/status mutation uses conditional `updateMany` guards.

**Tech Stack:** Node ≥20, Express 5, Prisma 6, SQLite (WAL), Zod, bcryptjs, Vitest + Supertest; React 19, Vite 6, react-router 7 (library mode), TanStack Query 5, Tailwind CSS 4.

## Global Constraints

- Read the spec `docs/superpowers/specs/2026-07-04-seapedia-design.md` §5 for all business-rule constants; they are restated inline where used.
- Money = integer rupiah (Prisma `Int`). **All rounding = `Math.floor`.**
- PPN = `floor(0.12 × (subtotal − discounts))`; delivery fee never taxed. `final_total = discounted_subtotal + ppn + delivery_fee`.
- Delivery fees/SLA: INSTANT Rp25000/1 day, NEXT_DAY Rp15000/2 days, REGULAR Rp10000/4 days.
- Error envelope everywhere: `{ "error": { "code": string, "message": string } }`. 400 validation, 401 unauth, 403 role/ownership, 404, 409 conflict.
- Order statuses (exact strings): `SEDANG_DIKEMAS`, `MENUNGGU_PENGIRIM`, `SEDANG_DIKIRIM`, `PESANAN_SELESAI`, `DIKEMBALIKAN`.
- Roles: `ADMIN`, `SELLER`, `BUYER`, `DRIVER`. Authorization = session's `activeRole`, verified server-side on every private endpoint.
- `schema.prisma` grows per level — never add future-level tables early (graded progressive history).
- Commits: conventional messages, per plan step; **never squash**. Suffix every commit body with `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>`.
- Backend tests: `cd backend && npx vitest run` (config sets `fileParallelism: false`, test DB `file:./test.db` reset between tests). All tests must pass before each commit.
- TypeScript strict mode both sides. No raw SQL anywhere (Prisma API only). No `dangerouslySetInnerHTML`.

return prisma.$transaction(async (tx) => {
  const cart = await loadCartOrThrow(tx, buyerId);            // 400 CART_EMPTY
  const address = await tx.address.findFirst({ where: { id: addressId, buyerUserId: buyerId } });
  if (!address) throw new ApiError(404, 'ADDRESS_NOT_FOUND', '...');
  const totals = computeTotals(subtotal(cart), deliveryMethod);
  for (const item of cart.items) {
    const r = await tx.product.updateMany({
      where: { id: item.productId, isDeleted: false, stock: { gte: item.quantity } },
      data: { stock: { decrement: item.quantity } } });
    if (r.count === 0) throw new ApiError(409, 'INSUFFICIENT_STOCK', `Stok ${item.product.name} tidak cukup`);
  }
  const debit = await tx.wallet.updateMany({
    where: { buyerUserId: buyerId, balance: { gte: totals.finalTotal } },
    data: { balance: { decrement: totals.finalTotal } } });
  if (debit.count === 0) throw new ApiError(409, 'INSUFFICIENT_BALANCE', 'Saldo wallet tidak cukup');
  // create order (status SEDANG_DIKEMAS, slaDeadline = addDays(virtualNow(), SLA_DAYS[method]) — Level 3 uses new Date(); virtualNow() swapped in at Level 6)
  // + orderItems with name/price snapshots + address snapshots
  // + statusHistory row { status: 'SEDANG_DIKEMAS', changedByRole: 'BUYER' }
  // + walletTransaction { type: 'CHECKOUT_CHARGE', amount: totals.finalTotal, orderId }
  // + clear cart items and set cart.storeId = null
});

export async function runOverdueSweep(): Promise<SweepResult[]> {
  const candidates = await prisma.order.findMany({ where: {
    slaDeadline: { lt: now() }, currentStatus: { notIn: ['PESANAN_SELESAI', 'DIKEMBALIKAN'] } } });
  const results = [];
  for (const order of candidates) {
    results.push(await prisma.$transaction(async (tx) => {
      const flip = await tx.order.updateMany({ where: { id: order.id, currentStatus: { notIn: ['PESANAN_SELESAI', 'DIKEMBALIKAN'] } }, data: { currentStatus: 'DIKEMBALIKAN' } });
      if (flip.count === 0) return { orderId: order.id, actions: ['skipped: status changed concurrently'] };
      const actions = ['status → DIKEMBALIKAN'];
      // history row { status: 'DIKEMBALIKAN', changedByRole: 'SYSTEM' }
      const refund = await tx.order.updateMany({ where: { id: order.id, isRefunded: false }, data: { isRefunded: true } });
      if (refund.count === 1) { /* wallet increment + REFUND tx row */ actions.push(`refund ${order.finalTotal}`); }
      const restock = await tx.order.updateMany({ where: { id: order.id, isStockRestored: false }, data: { isStockRestored: true } });
      if (restock.count === 1) { /* increment each orderItem.quantity back onto product.stock */ actions.push('stock restored'); }
      if (order.voucherId) { const v = await tx.order.updateMany({ where: { id: order.id, isVoucherRestored: false }, data: { isVoucherRestored: true } });
        if (v.count === 1) { /* voucher.usageRemaining increment */ actions.push('voucher quota restored'); } }
      await tx.deliveryJob.updateMany({ where: { orderId: order.id, status: { in: ['AVAILABLE', 'TAKEN'] } }, data: { status: 'CANCELLED', driverEarning: 0 } });
      return { orderId: order.id, actions };
    }));
  }
  return results;
}
